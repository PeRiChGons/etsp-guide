# Recursos visuales

Las imágenes públicas de la guía se organizan según el [plan de recursos visuales](../../docs/PLAN-DE-RECURSOS-VISUALES.md).

## Estado de una imagen

- `pendiente`: todavía no existe un recurso válido.
- `extraído`: se ha separado de una captura o fotograma.
- `revisado`: nombre, forma y uso han sido comprobados.
- `publicado`: se utiliza en una página de la guía.

Los vídeos y capturas originales de investigación no se guardan aquí. Esta carpeta contiene únicamente recursos optimizados para la web y sus fichas de procedencia.

## Nombre de archivo

```text
tipo-nombre-variante-v01.ext
```

Ejemplos:

```text
icono-dark-phoenix-v01.png
retrato-dragon-lancer-v01.webp
ficha-sistema-spirit-v01.webp
objeto-basic-cleansing-pill-v01.webp
habilidad-sky-piercer-v01.webp
fondo-seccion-spirit-v01.webp
decoracion-marco-dorado-v01.png
```

No se deben introducir estadísticas, nombres o efectos dentro de una imagen si el dato debe poder editarse o traducirse.
## Primer lote visual

- `backgrounds/spirit-section-bg.png`: fondo decorativo original para la seccion de espiritus. No contiene datos del juego.
- `spirits/zhuyuan-hero-render.png`: render conceptual generado a partir de una captura de referencia de Zhuyuan. Se usara como ilustracion de detalle y no sustituye a la captura real del juego.

Los renders generados deben mostrarse como apoyo visual. Los nombres, estadisticas, rarezas, habilidades y efectos seguiran estando en HTML para poder verificarlos, editarlos y traducirlos.

## Recursos visuales generados para la interfaz

- `backgrounds/etsp-hero-atmosphere.png`: fondo panoramico conceptual con un personaje y elementos de fantasia. Es decorativo y no representa una pantalla oficial del juego.
- `backgrounds/class-gallery.png`: composicion panoramica provisional para destacar las cuatro clases en la portada y en los accesos directos. Los colores son una guia visual, no una afirmacion sobre estadisticas o roles.

Estos recursos se usan para dar identidad a la web. La informacion jugable se mantiene siempre en texto, tablas y capturas verificables.
